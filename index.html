Link User to Tenant

1
Select the user
Find or create the DFRNT staff member in the Users list
SELECT UserId FROM [User]
WHERE Email = 'user@dfrnt.com'
2
Choose the tenant
Select which tenant environment to grant access to
SELECT TenantId FROM Tenant
WHERE Code = 'ts-us-prod'
3
Set the role
Role is stored as a TenantUserSetting row
-- Admin / Operator / ReadOnly
4
Insert the Master Controller link
Two inserts in the Master Controller DB — TenantUser + TenantUserSetting
INSERT INTO TenantUser (TenantId, UserId)
VALUES (42, 7);

INSERT INTO TenantUserSetting
(TenantId, UserId, SettingName, SettingValue)
VALUES (42, 7, 'Role', 'Operator');
✓
Create tblUser in the tenant's own DB ⚠️ Easy to miss
AdminManager reads StaffId from the auth cookie and looks it up in the tenant's own tblUser. Without this row, login fails with "User not found."
-- Run against TARGET tenant's DB
INSERT INTO tblUser (
  UserName, FullName, Password,
  UserGroupId,
  StaffId,        -- must match cookie claim
  Active, InternetAccess, CourierSecurity,
  Created, CreatedBy, LastModified, LastModifiedBy
) VALUES (
  'george', 'George Hampton', 'placeholder',
  @AdminGroupId,
  @StaffIdFromUrgentDB,
  1, 1, 0,
  GETDATE(), 'DFRNT', GETDATE(), 'DFRNT'
);
💡 Get StaffId first: SELECT StaffId FROM tblUser WHERE FullName = 'George Hampton' — run this against Urgent Couriers DB

How switching works
1.JWT token includes tenantId claim + list of all accessible tenantIds
2.User clicks switcher → selects target tenant
3.App calls POST /switch-tenant — no re-authentication
4.New JWT issued with updated tenantId, updates CurrentTenantId
5.App reloads data using new tenant's DB connection string
